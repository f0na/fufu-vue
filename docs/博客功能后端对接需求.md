# 博客功能后端对接需求文档

## 概述

前端需要实现博客/文章功能，包括文章列表、文章详情、归档页面。请后端根据以下需求设计数据结构和 API 接口。

---

## 一、数据结构需求

### 1.1 文章表 (posts)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string (UUIDv7) | ✓ | 主键 |
| title | string (max: 200) | ✓ | 文章标题 |
| slug | string (max: 200) | ✓ | URL slug，唯一索引，用于 SEO 友好 URL |
| summary | string (max: 500) | | 摘要/简介，用于列表展示 |
| content | text | ✓ | Markdown 格式文章内容 |
| cover | string (max: 500) | | 封面图 URL |
| tags | json/string[] | | 标签数组，如 `["Vue", "前端"]` |
| category | string (max: 50) | | 分类名称 |
| status | enum | ✓ | 状态：`published`(已发布) / `draft`(草稿) / `hidden`(隐藏) |
| view_count | integer | ✓ | 浏览数，默认 0 |
| like_count | integer | ✓ | 点赞数，默认 0 |
| comment_count | integer | ✓ | 评论数，默认 0 |
| is_top | boolean | ✓ | 是否置顶，默认 false |
| allow_comment | boolean | ✓ | 是否允许评论，默认 true |
| published_at | timestamp | | 发布时间（草稿可为空） |
| created_at | timestamp | ✓ | 创建时间 |
| updated_at | timestamp | ✓ | 更新时间 |

**索引建议：**
- `slug` 唯一索引
- `status` 索引
- `published_at` 索引（用于排序）
- `tags` JSON 索引（如数据库支持）
- `category` 索引

### 1.2 文章点赞记录表 (post_likes) - 可选

用于防止重复点赞（IP 或用户维度）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✓ | 主键 |
| post_id | string | ✓ | 文章 ID，外键 |
| user_id | string | | 用户 ID（登录用户） |
| ip | string | | 访客 IP（未登录用户） |
| created_at | timestamp | ✓ | 点赞时间 |

**注意：** 前端也会用 localStorage 记录已点赞文章，但后端建议也做防重。

---

## 二、API 接口需求

### 2.1 公开接口

#### 2.1.1 获取文章列表

```
GET /posts
```

**Query 参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | integer | | 页码，默认 1 |
| per_page | integer | | 每页数量，默认 10，最大 50 |
| tag | string | | 按标签筛选 |
| category | string | | 按分类筛选 |
| keyword | string | | 搜索关键词（标题、摘要） |
| year | integer | | 按年份筛选 |

**响应：**
```json
{
  "items": [
    {
      "id": "abc123",
      "title": "文章标题",
      "slug": "article-slug",
      "summary": "文章摘要...",
      "cover": "https://example.com/cover.jpg",
      "tags": ["Vue", "前端"],
      "category": "技术",
      "view_count": 100,
      "like_count": 10,
      "comment_count": 5,
      "is_top": false,
      "published_at": "2026-04-07T10:00:00Z",
      "created_at": "2026-04-07T10:00:00Z",
      "updated_at": "2026-04-07T12:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 10,
    "total": 100,
    "total_pages": 10
  }
}
```

**业务规则：**
- 只返回 `status = published` 的文章
- 置顶文章排在最前面
- 按 `published_at` 倒序排列

---

```
GET /posts/:id_or_slug
```

**路径参数：**
- `id_or_slug`: 文章 ID 或 slug

**响应：**
```json

  "slug": "article-slug",
  "summary": "文章摘要...",
  "content": "# 标题\n\n文章内容...",

  "tags": ["Vue", "前端"],
  "category": "技术",
  "status": "published",
  "view_count": 101,
  "like_count": 10,
  "comment_count": 5,
  "is_top": false,
  "allow_comment": true,
  "published_at": "2026-04-07T10:00:00Z",
  "created_at": "2026-04-07T10:00:00Z",
  "updated_at": "2026-04-07T12:00:00Z",
  "toc": [
    {
      "id": "heading-1",
      "text": "一级标题",
      "level": 1,
      "children": [
        {
          "id": "heading-1-1",
          "text": "二级标题",
          "level": 2,
          "children": []
        }
      ]
    }
  ],
  "recommended_posts": [
    {
      "id": "def456",
      "title": "推荐文章1",
      "slug": "recommended-1",

      "summary": "...",
      "tags": ["Vue"],
      "published_at": "..."
    }
  ]

```

**业务规则：**
- 访问时 `view_count` +1
- 如果文章 `status != published`，返回 404（或对管理员返回）
- `toc` 目录结构由后端解析 Markdown 标题生成，或前端生成
- `recommended_posts` 推荐 3-5 篇相关文章（同标签或同分类）

#### 2.1.3 获取归档列表

```
GET /posts/archive
```

**Query 参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| year | integer | | 按年份筛选 |
| tag | string | | 按标签筛选 |

**响应：**
```json
[
  {
    "year": 2026,
    "months": [
      {
        "month": 4,
        "posts": [
          {
            "id": "abc123",
            "title": "文章标题",
            "slug": "article-slug",
            "published_at": "2026-04-07T10:00:00Z"
          }
        ]
      },
      {
        "month": 3,
        "posts": [...]
      }
    ]
  },
  {
    "year": 2025,
    "months": [...]
  }

```

**业务规则：**
- 按年倒序、月倒序排列
- 只返回 `status = published` 的文章
- 返回精简字段（id, title, slug, published_at）

---

#### 2.1.4 点赞文章

```
POST /posts/like
```

**请求体：**
```json
{
  "post_id": "abc123"
}
```

**响应：**
```json
{
  "liked": true,
  "like_count": 11
}
```

**业务规则：**
- 同一用户/IP 对同一文章只能点赞一次
- 如果已点赞，可以取消（toggle 逻辑）或忽略
- 返回最新点赞数

### 2.2 管理接口（需要管理员权限）

#### 2.2.1 获取文章列表（管理员）

```
GET /admin/posts
```

**Query 参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | integer | | 页码 |
| per_page | integer | | 每页数量 |
| status | string | | 按状态筛选：published/draft/hidden |
| tag | string | | 按标签筛选 |
| category | string | | 按分类筛选 |
| keyword | string | | 搜索关键词 |

**响应：** 同公开接口，但返回所有状态的文章

---

#### 2.2.2 创建文章

```
POST /admin/posts
```

**请求体：**
```json
{
  "title": "文章标题",
  "slug": "article-slug",
  "summary": "文章摘要",
  "content": "# 标题\n\n内容...",
  "cover": "https://example.com/cover.jpg",
  "tags": ["Vue", "前端"],
  "category": "技术",
  "status": "published",
  "is_top": false,
  "allow_comment": true,
  "published_at": "2026-04-07T10:00:00Z"
}
```

**业务规则：**
- `slug` 如果不传，自动生成（可用日期+随机字符串，如 `20260407-abc123`）
- `slug` 必须唯一，重复时返回错误
- `status = published` 时，`published_at` 默认为当前时间

---

#### 2.2.3 更新文章

```
PATCH /admin/posts/:id
```

**请求体：** 同创建，所有字段可选

---

#### 2.2.4 删除文章

```
DELETE /admin/posts/:id
```

---

#### 2.2.5 切换文章状态

```
PATCH /admin/posts/:id/status
```

**请求体：**
```json
{
  "status": "published"
}
```

---

#### 2.2.6 切换文章置顶

```
PATCH /admin/posts/:id/top
```

**请求体：**
```json

  "is_top": true
}
```

## 三、评论系统集成

现有评论系统需要支持文章评论。

### 3.1 修改 CommentTarget 类型

```typescript
// 现有
export type CommentTarget = 'gallery' | 'bangumi'

// 改为
export type CommentTarget = 'gallery' | 'bangumi' | 'post'
```

### 3.2 评论 API 无需修改

现有的 `get_comments` 和 `create_comment` 接口通过 `target_type` 和 `target_id` 区分目标，只需前端传入 `target_type = 'post'` 即可。

## 五、补充说明

### 5.1 Slug 生成规则（前端处理）

**生成流程：**
1. 用户输入标题
2. 前端调用百度翻译 API → 返回英文
3. 转为 slug 格式（小写 + 连字符）
4. 用户可手动修改
5. 提交时后端检查唯一性，重复则追加随机后缀

**示例：**
| 标题 | 翻译结果 | Slug |
|------|---------|------|
| 我的第一篇博客 | my first blog | `my-first-blog` |
| Vue3 入门教程 | Vue3 intro tutorial | `vue3-intro-tutorial` |
| 深入理解 JavaScript | deep understanding of JavaScript | `deep-understanding-of-javascript` |

**技术方案：**
- 百度翻译 API（高级认证后 100万字符/月）
- 降级策略：API 失败时用 `pinyin-pro` 本地转拼音

**后端职责：**
- 检查 slug 唯一性
- 重复时自动追加随机后缀，如 `my-blog-abc12`

### 5.2 目录 (TOC) 生成

两种方案：
1. **后端生成**：解析 Markdown 内容，提取标题生成 TOC 结构
2. **前端生成**：后端只返回原始 Markdown，前端用 markdown-it 插件生成

建议后端生成，可减少前端计算量。

### 5.3 推荐文章算法

简单方案：
- 同标签文章，按发布时间倒序取 3-5 篇
- 如果同标签文章不足，补充同分类文章

## 六、接口响应格式统一

请确保所有接口遵循项目现有的响应格式：

```json
// 成功响应
{
  "code": 0,
  "message": "success",
  "data": { ... }
}

// 分页响应
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [...],
    "pagination": {
      "page": 1,
      "per_page": 10,
      "total": 100,
      "total_pages": 10
    }
  }
}

// 错误响应
{
  "code": 10401,
  "message": "参数缺失",
  "data": null
}
```
